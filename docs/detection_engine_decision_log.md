## A. 設計判断（Decision Records）

### D-001 検知エンジンを唯一の対外契約（DefectMap/NoiseProfile）に閉じる
- **決定**: 除去・UI・品質評価は `contracts.py` にのみ依存させる。
- **理由**: ユーザー最重要要件「検知を中核とするモジュール構成」「AI 差し替え・
  新アルゴリズムのプラグイン化・他ホスト移植・評価連携の拡張性」を単一の
  依存規律で満たす。検知器を差し替えても契約が同じなら後段は無変更。
- **代替案と不採用理由**: 各除去段が検知器の内部表現に直接触る案 → 密結合で
  拡張不能。却下。

### D-002 2系統（インパルス／持続）に分岐（ダストサーベイ §0.2）
- **決定**: ダストは時間系（SDI/ROD）、スクラッチは空間射影＋持続性追跡。
- **理由**: ［事実］ダスト=時間的外れ値、傷=時間的持続。両者に共通の主力手法は
  存在しない（サーベイ §8.1 有効性マトリクスの裏付け）。単一検知器で両狙いは不可。

### D-003 リファレンス実装は numpy/scipy のみ（OpenCV 不使用）
- **決定**: 検知コアを numpy/scipy に限定。cv2 は実素材 I/O のみ（CLI）。
- **理由**: ［考察］(1) OFX 非依存コアの原則を Python 段でも徹底、(2) scipy.ndimage
  の分離可能フィルタ・モルフォロジー・label は C++/OpenCV に 1:1 対応でき移植が
  素直、(3) 依存が軽く CI・可搬性が高い。
- **トレードオフ**: 既存 prototype（cv2 依存）とは別モジュール。統合は将来検討。

### D-004 適応しきい値 T=k·σ(x,y) を全検知器の共通基盤に
- **決定**: `NoiseProfile.sigma_map()` を SDI/ROD/Top-Hat/射影に供給。
- **理由**: ［事実/§5.1］ダスト検知精度はノイズ推定精度に律速。固定しきい値は
  グレイン強度で破綻（§2.1）。ノイズプロファイルを両プラグイン統合の急所とする。

### D-005 スクラッチ確定は「持続性＋信頼度」の二重ヒステリシス
- **決定**: `min_persistence=5` かつ `min_confidence=0.7` を既定。
- **理由**: ［事実/§4.2］「数フレーム未満しか持続しない検知は棄却」が誤検知抑制の
  最重要ルール。反復ダストが同一列に偶然並ぶ短寿命トラックを除去。
- **トレードオフ ［考察］**: 5フレーム未満しか出ない実スクラッチは見逃す。正確性
  （＝誤検知低減）優先の既定。短寿命傷はライブ仮検知→再スキャンで確定する設計
  （§9.1 B4）に委ねる。パラメータはユーザー可変にする予定。

### D-006 ダスト検知は空間∧時間の AND ゲート
- **決定**: `require_spatial_support=True`。SDI/ROD と Top-Hat 候補の AND。
- **理由**: ［考察/§9.1 B2］動き補償誤差起因の偽ダスト（時間的には外れ値だが空間的
  にダスト形状でない）を空間候補との AND で除去。実測で適合率 1.0 を達成。

## B. 試行錯誤・エラー記録（Tried & Failed）

### T-001 ダスト「画素 IoU 再現率」で低値（0.33）→ 指標の誤り
- **事象**: 初回、ダスト再現率が 0.33 と低く見えた。
- **原因**: 評価で GT マスクを 2px 膨張させ、検知は円盤コアのみのため、膨張リング
  分が未検知としてカウントされ画素再現率が構造的に頭打ち。
- **対処**: サーベイ §0.3 のインスタンス層の趣旨に沿い、**インスタンス級再現率**
  （各 GT ダスト塊が検知インスタンスで捉えられたか）＋画素適合率に変更。
  → 実際の検知は 8/8=100%（漏れなし）だったと判明。
- **教訓**: 疎な小欠陥は画素 IoU より instance hit で測る。除去マスクの膨張は
  評価指標と別問題。

### T-002 内容の垂直帯/段差がスクラッチとして誤検知
- **事象**: 静止シーンの垂直帯（柱・壁の縁を模した構造）が縦傷として大量に確定。
- **試行1（失敗）**: 帯を細くする → 傷と同幅で弁別不能。
- **試行2（部分成功）**: 左右フランクの単点比較で ridge/edge 判定 → ノイズに弱く
  実傷まで棄却、recall 低下。
- **試行3（成功）**: **FWHM（半値全幅）で弁別**。元射影 p の山頂から左右へ半値まで
  の距離を測り、片側が落ちない（プラトー＝段差）か合計幅が上限超（＝広い帯）なら
  棄却。§4.3 断面モデル / §1.4 Hessian の R_B（ブロブ/線弁別）の 1D 版。
  → 内容帯の誤検知 0、実傷 recall 1.0 を両立。
- **教訓**: 「線か段か」は残差（p−median）でなく**元射影の山の幅**で見るのが頑健。

### T-003 合成データが傷を内容帯の真上に配置し弁別不能ケースを自作
- **事象**: ランダム配置で GT スクラッチが内容帯（x∈[80,90]）に重なり、原理的に
  弁別不能な病的ケースを生成、recall が下がった。
- **対処**: synth で傷を帯近傍・傷同士の近接から離して配置。
- **教訓 ［考察］**: 「傷が内容エッジの真上」は静止シーンでは動き非追従性なしに
  弁別不能（§2.5）。評価データはこの病的重なりを避け、弁別可能な設定で測る。
  実素材ではこの重なりは手動レビュー（自動90%＋手動10%）に回す想定。

### T-004 偶数個メディアンの定義差（既存 C++ 移植の教訓、README より）
- **記録**: 既存 analysis/ の移植で、偶数個中央値を np.median（中央2値平均）に
  合わせないとダスト時間的単発性条件が過度に厳しくなり IoU=0 になった実績あり。
  本エンジンの C++ 移植でも同じ罠に注意（temporal median / ROD の順位統計）。
