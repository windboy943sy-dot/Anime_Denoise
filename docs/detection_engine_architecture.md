## 0. このプロジェクトで何を作るか（最重要要件の再確認）

作るのは「ノイズ・ダスト・スクラッチを除去するプラグイン」**ではない**。
最終目標は **「映像中のノイズ・ダスト・スクラッチを高精度に検知・分類・解析し、
その解析結果を利用して、各除去機能を独立して制御できる次世代 OpenFX
映像復元プラットフォーム」** である。

したがって本設計の中核は **検知・解析エンジン**であり、除去（Phase 2-4）は
その出力契約（`DefectMap` / `NoiseProfile`）**だけ**に依存する後段モジュール
として位置づける。この依存方向を崩さないことが拡張性の生命線である。

### 設計優先順位（ユーザー指定・厳守）
1. 検知精度 → 2. 誤検知率の低減 → 3. 再現性 → 4. 保守性 →
5. 拡張性 → 6. GPU 最適化 → 7. リアルタイム性能

**速度より正確性を優先**。Phase 1 では除去を一切行わない。

```
    Hybrid/AI)  Spatial/AI)   AI補完)
```

### 中核テーゼ（ダストサーベイ §0.2）［事実の統合］
- **ダスト（blotch）は時間的外れ値** → 時間解析（SDI/ROD）が最強の検知軸。
- **スクラッチは時間的に持続** → 時間差分では消える。空間・射影・形状が主軸で、
  時間軸は「持続による確証」に使う。
- ゆえに検知は **必ず2系統**（インパルス系統／持続系統）に分岐させる。
  単一検知器で両方を狙う設計は原理的に筋が悪い。

### カスケード原則（ダストサーベイ §7.1）［事実］
**高感度候補生成 → 高特異度検証**。各段は前段の候補のみ処理するため、後段が
重くても総コストは小さい（候補は通常全画素の ≪1%）。

## 2. モジュール構成と責務

| モジュール | 責務 | 主な根拠（サーベイ章節） |
|---|---|---|
| `contracts.py` | `DefectMap` / `DefectInstance` / `NoiseProfile` / 種別 enum。**唯一の対外契約** | ダスト §0.3, ノイズ §0.1 |
| `noise_profile.py` | ノイズ4軸推定（強度依存σ / 空間相関 / 時間σ・FPN / クロマ）と支配モデル分類 | ノイズ §1.1,1.2,1.7,3.1,4.1 |
| `spatial.py` | 空間候補生成: DoG/LoG・White/Black Top-Hat・Hessian vesselness | ダスト §1.1,1.2,1.4 |
| `temporal.py` | 時間ダスト検知: SDI・ROD・時間メディアン乖離・シーンチェンジ・近傍多数決 | ダスト §2.1,2.2,2.3 |
| `instances.py` | 連結成分化・形状特徴・ルールベース分類 | ダスト §1.3,4.5 |
| `scratch.py` | 帯別垂直射影・ridge/edge 弁別・1D 追跡（持続性ヒステリシス） | ダスト §4.2,4.3,2.5 |
| `analyzer.py` | オーケストレータ（2系統×2パス、カスケード、診断） | ダスト §7.1,9.1 |
| `visualize.py` | 検知のみオーバーレイ・確率ヒートマップ | ダスト §9.1 B7 |
| `synth.py` | GT 付き合成欠陥クリップ（精度検証用） | ダスト §6.1 |

`contracts` にのみ依存する。検知アルゴリズムの差し替えは契約を保てば後段に
波及しない（拡張性要件の担保）。

## 3. 出力契約（拡張性の要）

### 3.1 `DefectMap`（ダストサーベイ §0.3）
二層構造:
- **画素層** `prob`(0..1) / `alpha`(半透明度) / `labels`(→インスタンスID)
  除去（インペイント）はこの層で動く。`alpha` は減算/ブレンド修復用（§7.6）。
- **インスタンス層** `instances: [DefectInstance]`
  誤検知抑制・分類・追跡・UI（個別 on/off）はこの層で動く。

`DefectInstance` の属性: 種別・bbox・重心・面積・細長度・円形度・主軸方向・
極性・コントラスト・半透明度・時間持続情報・信頼度・**検知器ソース**
（どの手法が寄与したか＝誤検知分析用）。

### 3.2 `NoiseProfile`（ノイズサーベイ §0.1）
4軸: (1) 強度依存 `sigma_by_bin` (2) 空間相関 `spatial_correlation_length`
(3) 時間特性 `temporal_sigma`/`has_fixed_pattern`/`flicker` (4) 色 `sigma_per_channel`。
**どの色空間で推定したか** `color_space` を必ず記録（img-processing 規律）。

`sigma_at(luma)` / `sigma_map(img)` で強度依存の**適応しきい値** T = k·σ(x,y)
を全検知器に供給する。ダストサーベイ §5.1「ダスト検知精度はノイズ推定精度に
律速される」ため、このプロファイルが両プラグイン統合設計の急所。

## 4. アルゴリズム採用理由・不採用理由・代替案

各手法の詳細比較はサーベイ本体（`reference/`）と `detection_engine_decision_log.md`
に記録。ここでは Phase 1 での採否結論を示す。

### 4.1 ノイズ推定
| 手法 | 採否 | 理由 |
|---|---|---|
| Immerkær（§1.2） | **採用**（初期値） | 最速・パラメータレス。エッジ間引きで masked 化 |
| 局所分散＋輝度ビン（§1.1/1.7） | **採用**（主） | 強度依存σを直接得る。Foi 法の簡易一般化 |
| 構造指向均質ブロック選別（§1.5 Amer-Dubois） | **採用**（前処理） | エッジ混入を減らす。方向別高域で均質判定 |
| 時間差分σ（§3.1） | **採用** | 静止領域で Var(d)/2。FPN 疑い判定にも使う |
| PCA/固有値法（§1.6 Liu/Chen） | **保留** | 単一画像最高精度だが反復選別が重い。v2 の高精度モード候補 |

### 4.2 ダスト（インパルス系統）
| 手法 | 採否 | 理由 |
|---|---|---|
| SDI（§2.1 Kokaram） | **採用**（主軸） | ダストの単一フレーム性に直接立脚。最も特異度が高い |
| ROD（§2.2 Nadenau-Mitra） | **採用**（AND 強化） | 順位統計でグレインに頑健。SDI と投票で特異度↑ |
| 時間メディアン乖離（§2.3） | **採用**（ROD 代替可） | 軽量・頑健。無差別置換はせず検知量としてのみ使う |
| 空間 Top-Hat/DoG（§1.1/1.2） | **採用**（AND ゲート） | 時間検知の動き誤差起因の孤立点を空間候補との AND で除去 |
| MRF 統合（§2.4 JOMBADI） | **不採用**（v1） | 反復最適化で重い。軽量版（近傍投票）を将来検討 |
| Mask R-CNN/YOLO（§6.2） | **不採用** | 小物体・極端アスペクト比に anchor/NMS が不利 |

### 4.3 スクラッチ（持続系統）
| 手法 | 採否 | 理由 |
|---|---|---|
| 帯別垂直射影（§4.2 Joyeux） | **採用**（主軸） | 縦傷に圧倒的にロバスト。射影で SNR √H 倍。費用対効果最良 |
| ridge/edge 弁別（§4.3/§1.4 の 1D 版） | **採用** | FWHM で「細い線」と「内容の帯/段差」を弁別（誤検知抑制の核） |
| 1D 追跡＋持続性ヒステリシス（§4.2/§2.5） | **採用** | 数フレーム未満の検知を棄却＝誤検知抑制の最重要ルール |
| Hessian vesselness（§1.4） | **実装済・補助** | 曲線傷・方向特徴。射影が苦手な斜め傷用（B 系統で活用予定） |
| Hough（§4.1） / ウェーブレット（§3.2） | **保留** | 射影法で大半カバー。斜め傷が問題化したら追加 |
| RPCA/スパース（§3.3） | **不採用**（v1） | SVD が重くリアルタイム不適。オフライン高精度モード候補 |

### 4.4 AI（適材適所の原則）
Phase 1 は **古典手法で「学習なしで動く土台」を完成**させる方針。AI は候補限定
（Top-Hat/SDI で全画素の ≪1% に絞る）→ CNN パッチ検証（§7.3）の増築構造で v2
に載せる。多チャンネル入力（生画像＋SDI 応答＋vesselness＋補償残差）で幾何・時間
の弁別原理を注入する設計（§6.1）。RTN 型時空間 AI（§6.3）は将来の主検知器候補。
**AI モデルは差し替え可能**（`contracts` を保つ限りエンジン内部の一部品）。

## 5. 誤検知抑制の3本柱（ダストサーベイ §7.1）と本実装での実現

| 柱 | 原理 | 実装箇所 |
|---|---|---|
| (a) 時間確認 | ダスト=単発 / 傷=持続の物理 | `temporal.detect_dust_temporal`（SDI∧ROD）／`scratch` の持続性 |
| (b) 動き非追従性 | 傷は実シーンの動きに従わない（Müller 2009 §2.5） | ridge/edge 弁別で内容の垂直構造を棄却。**大域動き速度ゲートは保留（§7）** |
| (c) 追跡持続数のヒステリシス | 数フレーム未満の検知を棄却 | `ScratchTracker.min_persistence=5` ＋ `min_confidence=0.7` |

補助:
- 適応しきい値 T=k·σ(x,y)（グレイン強度依存）— 固定しきい値は破綻する（§2.1）
- 近傍多数決（3×3 過半）でノイズ起因の点誤検知を除去（§2.1）
- シーンチェンジ検出でカット跨ぎフレームのダスト検知を無効化（§2.1、必須）
- 空間∧時間の AND ゲート（動き補償誤差起因の偽ダスト除去、§9.1 B2）

## 6. 実測結果（本リポジトリ）［実測 2026-07-19］

合成 GT クリップ（180×240×12〜15f、AWGN σ=0.015〜0.03、白/黒ダスト・持続縦傷・
内容の垂直帯/段差を含む）に対する `prototype/tests/test_detection_engine.py` の測定:

| ケース | ノイズσ推定誤差 | ダスト適合率 | ダスト再現率(instance) | 傷再現率 | 傷誤検知トラック |
|---|---|---|---|---|---|
| static_lowNoise (σ0.015) | 5.5% | 1.000 | 1.000 | 1.00 | **0** |
| static_midNoise (σ0.030) | 1.7% | 1.000 | 0.988 | 1.00 | **0** |
| denser (σ0.020, dust×12, 傷×3) | 3.1% | 1.000 | 1.000 | 1.00 | **0** |

- ノイズσ推定は相対誤差 <6%（`estimate_noise_profile`）。
- ダスト適合率 1.0（＝誤検知ゼロ）は「正確性優先」要件に合致。
- 傷は全 GT を検出しつつ、内容の垂直帯/段差を **0 件**で棄却（ridge/edge 弁別が有効）。

**注意 ［実測の限界］**: これは合成データの測定であり、実フィルム素材の性能では
ない。実素材評価（グレイン・実傷テンプレート）は `reference` サーベイ付録 C の
課題であり、本リポジトリの `test-assets/raw/`（25本）での評価が次段。

## 7. 既知の限界・保留事項（詳細は decision_log）

1. **動き補償（MC）未実装**: `temporal` は静止前提（恒等補償）。動きのある実素材
   では SDI が破綻する（§2.1 短所）。粗ブロックマッチング MC が次の最優先実装。
   → それまで本エンジンは「静止シーン or MC 供給済み入力」で正確。
2. **動き非追従性ゲート未実装**: 静止シーンの細い内容縦線は ridge/edge 弁別と
   持続性で大半棄却できるが、原理的完全弁別には大域動き速度ゲート（§2.5 Müller）
   が要る。実装は MC 導入後。
3. **斜め・曲線傷**: 現状の垂直射影は縦傷特化。Hessian vesselness は実装済みだが
   B 系統の分類にまだ接続していない。
4. **カビ・半透明汚損**: 持続性解析（§2.5）と α 推定の枠は用意したが専用検知器は未。
5. **PNG I/O**: リファレンス可視化用の stdlib 実装。本番は cv2/OFX 経由。

## 8. 拡張ポイント（将来機能の受け口）

ユーザー要件「拡張性を考慮したアーキテクチャ」に対する具体的な受け口:

| 追加したい機能 | 受け口 |
|---|---|

| AI モデルの差し替え・追加 | 検知器を `DetectorSource.CNN` として候補検証段に差し込む。ONNX Runtime EP 切替（§9.3） |

| 他 OFX ホスト移植 | エンジンは OFX 非依存。ホスト差分は境界層のみ（cpp_port_design.md） |

| リアルタイム/GPU 最適化 | 密カーネル群（DoG/TopHat/SDI/ROD）は近傍共有でカーネル融合（§9.3）。疎処理以降は候補駆動 |

## 9. C++/OpenFX 移植方針（要点、詳細は cpp_port_design.md）

- **2層構造**: OFX 境界層 ＋ OFX 非依存 C++ コア。コアは合成 GT で単体テスト（CI）。
- **numpy/scipy → C++/OpenCV 対応**:
  `gaussian_filter`→分離ガウシアン、`white/black_tophat`→`cv::morphologyEx`、
  `median_filter`→`cv::medianBlur`/ヒストグラム法、`ndimage.label`→
  `cv::connectedComponentsWithStats`、Hessian 固有値→閉形式（自作カーネル）。
- **時間アクセス**: t±1（可能なら t±2）fetch。ホスト供給不可時の縮退チェーン:
  3フレーム SDI → 片側差分（感度低下明示）→ 空間のみ。Resolve 実挙動は要実測（付録 C-4）。
- **決定性**: 同一入力→同一出力。ライブ仮検知はプレビューのみ、確定レンダは台帳参照。
- **GPU**: 密カーネルは CUDA/Metal/OpenCL 三重実装（コアロジック共通ヘッダ化）。
  CNN 検証は ONNX Runtime の EP 切替。OpenCL 環境は純古典モードにグレースフルデグラデーション。
