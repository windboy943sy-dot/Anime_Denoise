# デノイズ手法調査と方式提案 — エッジを甘くせず最大限きれいにする（2026-07-12）

Phase 3 着手にあたり、既存のデノイズ・フィルム復元手法を調査し、
本プロジェクトの構造（保持グループ＝同一絵柄のフレーム集合が既知）を
最大限活かす方式を設計した。実装は `prototype/denoise/`。

---

## 1. 手法調査の要約

### 1.1 空間フィルタ系（単一フレーム内で完結）

| 手法 | エッジ保持 | 除去力 | 備考 |
|---|---|---|---|
| ガウシアン／メディアン | ✗ 甘くなる | 中 | 論外（比較基準） |
| バイラテラル／ガイデッドフィルタ | △ 強度を上げると甘くなる | 中 | 軽量。弱く使う分には安全 |
| NLM（Non-Local Means） | ○ | 中〜高 | 自己相似性を利用。アニメのベタ塗り＋反復構造と相性が良い |
| BM3D | ◎ | 高 | 空間系の実質的な最高峰。ブロックマッチング＋協調フィルタリング |
| Wavelet／DCT縮退 | △ | 中 | リンギングが線画に出やすい |

- どんなに優れた空間フィルタでも「グレインと本物の微細ディテールの区別」は
  原理的に確率的で、強くかけるほど必ずどこかが甘くなる。
  → **空間フィルタは主役にしない**というのが本方式の第一原則。

### 1.2 時間方向・動画系

| 手法 | 特徴 |
|---|---|
| 時間平均／中央値 | 同一内容のフレームが揃えば理論上ノイズ除去の最適解（σ/√N）。エッジ劣化ゼロ |
| MVTools系（SMDegrain, TemporalDegrain2） | 動き補償＋時間統合。AviSynth/VapourSynth の定番。動きベクトル誤りでにじみが出る |
| VBM3D／VBM4D | 時空間ブロックマッチング。動画デノイズの古典的最高峰。重い |
| QTGMC系 | インターレース処理兼用。ノイズ除去を上げるとぼけるとの報告多数 |

- ポイント：時間統合の品質は「同一内容フレームをどれだけ正確に集め / 合わせられるか」で決まる。
  一般動画ではここに動き推定の誤りリスクが入るが、
  **コマ打ちアニメでは Phase 1 の保持グループ検出がこの問題をほぼ消してくれる**
  （動き推定ではなく「同一絵柄の判定」なので誤りモードが穏やか）。

### 1.3 フィルム復元系（ダスト・傷・ブロッチ）

- SDIa（Spike Detection Index）：前後フレームとの二乗差の最小値を閾値判定。単純だが誤検出が多い
- ROD / SROD（Ranked Order Difference）：前後フレーム画素のランク統計で判定。誤検出が減る
- 傷（線スクラッチ）：複数フレームで同位置に持続する縦線を検出（A Contrario法、Hough、ガボール）
- 補修：多段メディアン、エッジ優先の補間、AR-Medianモデル
- 共通の課題として「動き補償誤りとグレインによる誤検出」が挙げられている
  → 本方式では保持グループ内は動き補償が不要なので、この誤検出源が最初から無い

### 1.4 AI系

- RTN「Bringing Old Films Back to Life」(CVPR 2022)・RRTN：劣化フィルムの
  総合復元。強力だが (1) 生成的な補完で「絵が変わる」リスク、(2) GPU必須で重い、
  (3) 写実データ学習のモデルはアニメの線画をノイズ扱いして潰す傾向
- waifu2x / Real-ESRGAN(anime)：アニメ特化の学習で線画保持は良好。
  ただし主目的はアップスケールで、フィルムグレイン除去はスコープ外
- 商用復元でも「アニメの線画・セル塗りを写実系AIが破壊する」問題は широко知られており、
  周到なマスクと人手確認が前提とされている
- → **AIは本方式の必須要素にしない**。将来のオプション（後述の第4層）に留める

## 2. 提案方式：「時間統合を主役、空間フィルタは脇役」の4層構成

原則：**エッジを守る最善の方法は、エッジに触れるフィルタを使わないこと。**
グレイン除去の主役を時間方向に置けば、空間方向（＝エッジのある方向）には
一切手を付けずに済む。動きがあって時間統合できない場所だけ、
線画を保護した空間フィルタで妥協する。

```
第1層  保持グループ内 時間統合（エッジ劣化ゼロの主役）
        ├─ サブピクセル位置合わせ（ウィーブ補正）← これを怠ると逆にエッジが甘くなる
        ├─ トリム平均で参照像R生成（外れ値＝ダストを自動排除）
        └─ グループ内フレーム数Nに応じ σ→σ/√N でグレイン減衰

第2層  大域動きのみのカット間 拡張統合（パン・ズーム）
        ├─ Phase 2 の大域ワープで隣接グループのRを現グループ座標へ変換
        ├─ トレランス残差がノイズ床以下の画素だけ統合に参加（安全側）
        └─ 実効Nが増える＝ほぼ静止のパン・ズームでも第1層並みの品質に

第3層  動領域の空間デノイズ（最後の手段・線画保護つき）
        ├─ Phase 2 の局所動きマスクで領域を限定
        ├─ NLMベース、強度はグループ内で実測したグレインσから自動設定
        └─ Canny線画マスク近傍は元画素にフォールバック（輪郭は完全保持）

第4層  （将来オプション）AI仕上げ
        └─ 第1〜3層の出力を入力に、残った欠陥のみアニメ特化モデルで補修。
           デフォルトOFF・Before/Afterプレビュー必須
```

### 2.1 第1層の要：統合「前」のサブピクセル位置合わせ

フィルムスキャンにはゲートウィーブ（フレームごとの±0.5px程度の位置揺れ・
±0.3°程度の回転揺れ）が実測で存在する。位置合わせせずに平均すると
**この揺れ幅ぶんエッジがにじむ**。ECC（MOTION_EUCLIDEAN）で
基準フレームへサブピクセル位置合わせしてから統合する。

実測（05_3coma_01.mov 末尾の7フレーム止め、2560x1920）：

| 処理 | エッジ鮮鋭度* | 平坦部ノイズσ |
|---|---|---|
| オリジナル | 162.9 | 1.96 |
| 位置合わせ**なし**で統合 | 158.6（劣化） | ≈0 |
| 位置合わせ**あり**で統合 | **170.5（向上）** | **0.000** |

*エッジ鮮鋭度＝Cannyエッジ上の平均Sobel勾配強度。
位置合わせ付き統合はノイズを完全除去しつつ、グレインがエッジを
揺らさなくなるぶん**オリジナルよりシャープになる**。

### 2.2 参照像Rはトリム平均（既定）

- median：外れ値耐性は最強だがN が小さいと量子化が残る
- **trimmed mean（上下25%落とし・既定）**：ダスト等の外れ値を落としつつ
  残りを平均するので、グレイン抑制と外れ値耐性の両立
- mean：ダストが平均に混入するので単体では使わない

### 2.3 欠陥検出はグループ内で完結（動き補償誤りが存在しない）

従来のフィルム復元（SDIa/ROD）は「前後フレームを動き補償してから差分」を
取るため、動き補償誤り＝誤検出だった。本方式は保持グループ内の
位置合わせ済みフレームと R の残差なので、この誤検出源が構造的に無い。

判定は4条件のANDで、いずれも実測で必要性を確認済み：

1. **振幅**：輝度残差 |Y(frame)−Y(R)| > k×グレインσ
   （σはグループ内残差のMADからロバスト推定。カットごとのグレイン量に自動適応）。
   ただし振幅だけではフィルムグレインの重い裾が大量に貫通する
   （実測：画面の13%が誤検出）ので、これ単独では成立しない
2. **時間的単発性**：同グループの**他フレーム**では同位置の残差が小さいこと。
   グレインは全フレームで揺らぎ続け、ダストはそのフレームにしか無い、という
   出現パターンの違い（設計提案書3.1節の表）をそのまま条件化したもの。
   これで誤検出が桁で減る（13%→0.1%以下）。
   傷（複数フレーム持続）はここで意図的に落ちる＝持続性ベースの別ロジックで扱う
3. **形状**：孤立したコンパクトな塊のみ。微小（<4px）→グレイン扱い、
   大面積（>0.08%）→絵の変化扱い、細長い成分→位置ずれ残差／傷として除外
4. **場所**：フィルム枠・パーフォレーション近傍（外周10%）は対象外。
   枠線はフレームごとに揺れるため恒常的な偽残差源になる（実測済み）。
   線画エッジ近傍は閾値2倍の保守的判定（5章）。エッジマスクは生フレームでなく
   **Rから取る**（グレインをエッジ扱いしないため）

補正はマスク画素のみRで置換（inpaintではなく同グループの実測値を使う）。

実測（texture_preservingモード、2560x1920＝1フレーム490万画素）：

| 素材 | フレームあたり検出画素 |
|---|---|
| クリーン素材の止め区間（誤検出＝少ないほど良い） | 25〜125 px（0.002%） |
| ダスト素材の静止区間・通常フレーム | 数十〜数百 px |
| ダスト素材のダストフラッシュのあるフレーム | 1,800〜6,400 px（明確なスパイク＝正検出） |

### 2.4 質感の扱い：2モード（設計提案書3.1節をそのまま実装）

- full_temporal_integration：R を出力。除去品質最大
- texture_preserving：出力は各フレーム自身。欠陥画素のみR置換、
  グレインの揺らぎは保持。`grain_reduction` >0 のときだけ
  線画保護つきNLM（第3層と同じもの）を弱く併用

### スキャンノイズ検出（FFT周期解析）の実測

走査方向平均プロファイルのFFTスペクトルから、局所メディアンの spike_factor 倍を
超えるスパイクを検出（絵柄由来の低周波は近傍ビンもまとめて大きいためスパイクに
ならない）。補正はスパイク成分のみ逆FFTで再構成して差し引くノッチ方式。
実測：クリーン素材で誤検出0（水平・垂直とも）、合成縞（周期12px・振幅2.5）を
周期12.0px・振幅2.525と正確に検出、補正で平均偏差 1.556→0.916。

### 傷・ラインノイズ検出の設計と実測

- **傷**：保持グループ内では傷は参照像Rにも入る（残差に出ない）ため、ダストと逆に
  **グループ間の持続性**で検出する。細い縦線応答（水平メディアン差＋垂直連続性）の
  グループ間最小値を取り、被覆率の高い列を候補にする。絵柄の縦線はカメラワークで
  動けば持続性テストで消える。静止カットでは原理的に区別できないため
  **候補の提示まで**とし、除去はユーザー確認前提（5章の誤除去回避）。
  実測：クリーン素材（ズーム）で誤検出0列、合成傷（x=1000）を被覆率100%で検出、
  Telea inpaint 補修後の残差はグレイン水準（1.15）
- **ラインノイズ**：行/列の輝度中央値プロファイルの外れ値検定＋**一様性チェック**
  （本物のラインノイズは行全体が同符号で一様にずれる。絵柄エッジはばらける）。
  実測：クリーンで誤検出0行、合成オフセット(+8/−6)を7.92/−5.95と推定、補正後残差0.00
- CLI：`run_defect_scan.py`（傷候補列＋グループ別ラインノイズのレポートJSONを出力）

### 第3層の接続（時間↔空間の連続ブレンド）

設計提案書3章の「時間方向重みと空間方向重みを連続的に変化させるブレンド方式」を
`blend_spatial_fallback` として実装。第2層が返す**実効Nマップ**を使い、
時間統合が効かなかった画素（N=1）にだけ空間NRを重み1で適用、Nが増えるほど
1/N で減衰させる。重みマップは空間的にぼかして分類境界のフリッカーを防ぐ。
空間NR自体が線画エッジ保護つきなので、どの重みでもエッジには触れない。
（`run_denoise.py --extend N --grain-reduction 0.5` で有効）

CLI：`run_denoise.py`（Phase 1 の JSON → デノイズ済み動画＋左右比較動画。
`--extend N` で第2層が有効になり、前後Nグループをスライディングウィンドウで統合）。
処理速度は 82フレームで約10分（Python・シングルプロセス）。プロトタイプとしては
許容し、速度は Phase 6 の C++ 移植・Phase 8 の GPU 化で解決する。

### 画素単位モーションガード（7-12追加：芝居消失の防止）

品質検証で、**Phase 1 の閾値以下の微細な芝居（口パク）が完全時間統合で
平均化されて消える**重大な副作用を発見（07_CharaOnly_03：71フレームが
1グループ化→口が全フレームで半透明のゴーストになった）。
往復する動きは累積ドリフト検査（端点比較）もすり抜ける。

対策として analyze_hold_group に画素単位のモーションガードを実装：

- 条件：**内部フレーム**（端±1を除く）の25%以上が max(6, 4σ) を超えて
  変動する画素。ダストは単発（1フレーム）なので該当せず共存できる
- フィルム枠・パーフォレーション領域は除外（枠はフレームごとに揺れるため
  誤検出の主成分だった。可視化で確認）
- 連結成分の面積≥300px のみ採用（グレインの重い裾の偶然の一致は小斑点）。
  CLOSE で膨らませる**前に**面積フィルタを掛けるのが重要
- ガード画素は統合せずフレーム自身の値を保持（grain_reduction>0 なら
  線画保護つき空間NRのみ適用）。重みはぼかして境界の不連続を防ぐ

検証：口パク素材で口領域の輝度変動幅 7.40→5.85 と芝居が保持され
（ガード無しでは2.90に潰れる）、真の止め・ダスト素材ではガード0画素
（誤発動なし）、ダスト検出も維持。

設計メモ：Rの勾配でガード成分を棄却する案は**不成立**（ガード領域のRは
ゴーストで汚染されており勾配判定が循環する）。時間軸（端フレーム集中か
内部分布か）と空間（面積・枠除外）の組み合わせが正解だった。

### 未解決の課題

### 統合安全ガード（7-13追加：低速連続動きによるエッジ二重化の防止）

DaVinci 実機テストで、02_Zoom_02（連続低速ズーム）にエッジ二重化・軟化が発生。
調査で判明した事実：

1. フレーム間の動きが検出閾値以下のまま累積する素材は、保持グループとして
   誤統合される（Phase 1 のドリフト検査は「全画面平均」なので、空などの
   平坦部に薄められてエッジの数pxのずれを見逃す）
2. さらに **Euclidean ECC がこの素材で誤収束**し、整っていたフレームを逆に
   ずらしていた（エッジ残差 4.3→11.5 に悪化）
3. MOTION_AFFINE への変更は不可：自由度過多でグレインに適合し、
   真の止めでも R を劣化させる（実測）
4. 位置合わせの採否を残差やRの鮮鋭度指標で自動判定する案は全滅：
   グレインが指標を支配し、ウィーブ（0.5px）の改善も誤ワープの悪化も見えない

最終解：**端点エッジ残差による統合可否判定**。グループの内側端点
（端±1を除く、局所歪み対策）同士の「エッジ画素上の」残差中央値が
max(6.0, 5σ) を超えるグループは統合禁止とし、フレーム自身を出力する。
実測7ケース（止め・口パク・ダスト＝可、低速ズーム3種＝不可）で全問正解。

限界の明示：統合禁止グループは第1層の恩恵を受けない（グレインが残る）。
連続動き素材の時間デノイズは第2層（ワープ受け入れ統合）の担当で、
CLI では対応済み・OFXプラグインは v2 で搭載予定。
